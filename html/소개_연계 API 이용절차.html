연계 API 이용 절차

이용기관과 민간 SaaS 기업이 민간 SaaS 연계공통기반을 이용하는 절차를 안내합니다.

민간 SaaS 연계공통기반 이용은 운영기관, 민간 SaaS 기업, 이용기관, 정보보유기관이 참여해 다음과 같은 절차로 진행됩니다.

1-1

사전준비

사업계획

1-1단계 사전 준비 중 사업 계획 단계의 도식화입니다.
사전 준비 중 사업 계획 단계는 다음과 같은 순서로 진행됩니다. 1. 운영기관은 이용기관에 민간 사스 연계공통기반 통합관리포털(이하 통합관리포털) 내 “공공용 민간 사스“ 메뉴를 통해 공공용 민간 사스 카탈로그를 제공합니다. 2.이용기관은 사전검토 안내의 공공부문 이용가능 사스 체크사항을 확인합니다. (국가공공기관 민간 클라우드 서비스 도입 가능 목록 및 공공용 민간 사스 카탈로그 확인) 3. 이용기관은 민간 사스 기업에 이용 희망 민간 사스 관련 협의를 요청합니다. 4. 민간 사스 기업은 이용기관의 협의 요청을 수신합니다. 5. 민간 사스 기업과 이용기관은 함께 서비스 내역, 비용, 이용 기간, 이용자 규모, 보안성 검토, 전자정부 연계 API 이용 여부 등을 협의합니다. 6. 민간 사스 기업과 이용기관은 통합관리포털 내 이용 가이드 등을 통해 연계 서비스 이용절차 조회를 하고 숙지합니다. 7. 운영기관은 민간 사스 기업과 이용기관에 통합관리포털 내 이용가이드 등을 통한 연계 서비스(전자정부 연계 API) 이용 절차를 안내합니다. 8. 이용기관은 협의 결정사항에 따른 사업 계획을 수립하고, 민간 사스 기업은 이를 지원합니다.

사전준비

회원가입

1-2단계 사전 준비 중 회원 가입 단계의 도식화입니다.
사전 준비 중 회원 가입 단계는 다음과 같은 순서로 진행됩니다. 1. 민간 사스 기업과 이용기관은 각각 통합관리포털에 회원가입합니다. 회원가입 시 업로드해야 하는 서류는 다음과 같습니다. 민간 사스 기업 필요 서류: 사업자 등록증 / 이용기관 필요서류: 공공기관 고유번호증, 사업자 등록증 2. 운영기관은 민간 사스 기업과 이용기관이 통합관리포털에 회원가입한 내용의 정보를 확인하고, 필요시에는 정보의 보완을 요청합니다. 3. 민간 사스 기업과 이용기관은 운영기관으로부터 정보의 보완을 요청받은 경우 이를 확인하고 정보를 수정합니다. 4. 운영기관은 민간 사스 기업과 이용기관의 통합관리포털 회원가입을 승인하고 사용자에게 일반회원 등급을 부여합니다. 5. 민간 사스 기업과 이용기관은 회원가입이 승인된 것을 확인합니다.(일반회원 등급) 일반회원이란, 통합관리포털 회원가입은 완료했으나 보안성 검토를 한 번도 완료하지 않은 사용자를 의미하여, 사전계획, 회원가입, 사전검토 기능만을 이용할 수 있습니다.

사전검토

1-3단계 사전 준비 중 사전 검토 단계의 도식화입니다.
1-3단계 사전 준비 중 사전 검토 단계는 다음과 같은 순서로 진행됩니다. 1. 민간 사스 기업은 사전준비 신청을 합니다. 사전준비 신청 시 입력이 필요한 정보는 민간 SaaS 정보, 이용기관 정보, 서비스 계약 정보, 이용예정 전자정부 연계 API, 서비스 이용계획서 입니다. 2. 이용기관은 사전준비 신청 정보를 검토 하고 승인합니다. 3. 이용기관은 국가정보원과 상위기관의 보안성 검토 의뢰를 수행합니다. 4. 이용기관은 운영기관에 보안성 검토 결과를 제출합니다. 필요한 제출 서류는 보안성 검토 결과서이며, 선택 입력 정보는 이용기관 유형 선택 정보입니다. 5. 운영기관은 이용기관이 제출한 보안성 검토 결과를 검토하고 승인합니다. 또한 민간 사스 기업과 이용기관 사용자에게 정식회원 등급을 부여합니다. 6. 민간 사스 기업과 이용기관은 담당자가 통합관리포털 정식회원 등급이 된 것을 확인합니다. 정식회원이란 최소 1건 이상의 보안성 검토 과정을 완료한 회원을 의미하며, 모든 기능 사용이 가능하고, API 이용 신청, 실제 API 호출 및 연계를 할 수 있습니다.

2-1단계 전자정부 연계 API 이용 신청 및 승인 중 전자정부 연계 API 이용 신청 단계는 다음과 같은 순서로 진행됩니다. 1. 민간 사스 기업은 통합관리포털에서 전자정부 연계 API 이용을 신청하고 관련 서류를 제출합니다. 이용 신청은 다음의 세가지 경우에 합니다. 전자정부 연계 API 신규 신청인 경우, 전자정부 연계 API 확대 신청인 경우, 이용 기관 확대 신청인 경우입니다. 이때 필요한 서류는 이용계약서 및 SLA입니다. 2. 이용기관은 통합관리포털에서 민간 사스 기업이 제출한 이용신청을 검토하고 확인합니다. 이 때, 확인 버튼을 반드시 클릭해야 합니다. 3. 운영기관은 민간 사스 기업이 이용신청하고 이용기관이 검토 및 확인한 전자정부 연계 API 이용 신청을 승인합니다. 4. 운영기관은 민간 사스 기업에 신청 승인을 통보하고 테스트 키를 배포합니다. 5. 민간 사스 기업은 신청 승인된 것을 확인하고 테스트 키를 수령합니다.

2-2단계 전자정부 연계 API 이용 신청 및 승인 중 연계 개발 및 테스트 단계는 다음과 같은 순서로 진행됩니다. 1. 민간 사스 기업은 테스트를 실시합니다. 2. 이용기관은 LDAP 허용 데이터를 선택합니다. 단, LDAP 연계 API 이용 시에 한하며, 민간 사스 기업의 API 인증 키 신청 전에 선택해야 합니다. 3. 민간 사스 기업은 통합관리포털에서 API 인증 키를 신청합니다. 단, LDAP 연계 API 이외의 타 연계 API 이용 시엔 테스트 이후 API 인증 키를 신청해야 합니다. 4. 운영기관은 민간 사스 기업의 API 인증 키 신청을 승인하고 민간 사스 기업에 인증 키를 발급합니다. 5. 민간 사스 기업은 민간 SaaS에 적용 개발합니다. 6. 민간 사스 기업은 운영기관에 서비스 개통 증명을 신청합니다. 7. 운영기관은 민간 사스 기업의 서비스 개통 증명을 확인합니다.
