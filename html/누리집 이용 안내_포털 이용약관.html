제1장 총칙

제1조(목적)

본 약관은 민간 SaaS 연계공통기반 통합관리포털(이하 “포털”이라 함)을 통해 전자정부 연계 API의 이용을 신청하고 승인하는 일련의 절차 및 주체별 준수 사항을 규정함을 목적으로 한다.

제2조(용어의 정의)

본 약관에서 사용하는 용어의 정의는 다음과 같으며, 본 약관에서 정의하지 않는 용어는 「전자정부법」 및 동법 시행령, 「클라우드컴퓨팅 발전 및 이용자 보호에 관한 법률」 및 동법 시행령에서 정한 정의나 일반적인 개념에 따른다.

  1. ① 포털이란 민간 SaaS 기업에 전자정부 연계 API를 제공하기 위해 이용신청, API 인증키 제공, 전자정부 연계 API 이용 모니터링 및 Q&A 등의 기능을 제공하는 것을 말한다.
  2. ② “전자정부 연계 APIˮ(이하 “연계 APIˮ이라 함)란 총괄기관(행정안전부)이 이용기관의 업무 수행상 필요한 행정정보를 API 등 형태로 개발하여 민간 SaaS에 제공하는 것을 말한다.
  3. ③ “이용기관ˮ이란 ｢전자정부법｣ 제2조 2항에 해당하는 행정기관(지자체 포함), 같은 조 3항에 해당하는 공공기관, 교육부 고시 제2024-2호 제12조에 따라 EPKI 인증서를 발급받은 EPKI 발급 대상기관을 말한다.
  4. ④ “민간 SaaS 기업ˮ이란 자체 클라우드컴퓨팅서비스에 전자정부 연계 API를 적용하여 이용기관에 민간 SaaS를 제공하는 업체를 말한다.
  5. ⑤ “정보보유기관ˮ이란 행정정보를 직무상 작성 취득하여 유지·관리하는 행정기관을 말한다.
  6. ⑥ “운영기관ˮ이란 한국지능정보사회진흥원(NIA)으로 전자정부 연계 API 개발·배포 및 연계 관리를 총괄기관(행정안전부)으로부터 위탁받아 수행하는 기관을 말한다.
  7. ⑦ “APIˮ란 Application Programming Interface의 약자로서 전자정부 연계 API 와 민간 SaaS 간 정보보유기관의 행정정보를 송수신하는 프로그램을 말한다.
  8. ⑧ “API 서비스ˮ란 정보보유기관이 행정정보를 민간 SaaS를 통하여 이용기관에 제공 할 수 있도록 API 방식으로 전자정부 연계 API에 등록·제공하는 서비스를 말한다.
  9. ⑨ “사용자”란 포털에 회원으로 등록하고 이 약관에 따라 한국지능정보사회진흥원(NIA)이 제공하는 서비스를 정상적으로 사용할 수 있는 권한을 부여받은 자를 말한다.
  10. ⑩ “휴면계정”이란 포털에 1년 이상 로그인하지 않거나, 보안성 검토 부적격 판정 후 API가 중단되어 1년 이상 경과한 계정을 말한다.

제3조(약관의 효력 발생 및 개정)

  1. ① 이 약관은 서비스 메뉴에 게시하여 공시함으로써 효력이 발생한다.
  2. ② 한국지능정보사회진흥원(NIA)이 필요한 경우 ｢약관의 규제에 관한 법률｣ 등 관련 법령에 위배되지 않는 범위에서 이 약관을 개정할 수 있으며, 약관을 개정한 경우에는 지체 없이 공지하여야 한다. 다만, 이용자의 권리 또는 의무에 관한 중요한 내용의 변경은 최소한 7일 이전에 공지한다.
  3. ③ 제2항에 따라 개정된 약관을 공지한 후 사용자가 약관의 효력발생일로부터 7일이 경과할 때까지 거부 의사를 표시하지 아니한 경우에는 약관의 변경 사항에 동의한 것으로 본다.

제2장 전자정부 연계 API의 이용

제4조(포털 서비스 범위)

  1. ① 포털 서비스 제공 범위는 다음 각 호와 같다.
    1. 1. 전자정부 연계 API 이용을 위한 회원의 등록·삭제 및 회원정보 관리
    2. 2. API 서비스를 위한 API 목록 및 기능 정보 제공
    3. 3. 전자정부 연계 API 접속을 위한 API Key 발급·관리
    4. 4. 현황 모니터링 및 통계 조회
    5. 5. 그밖에 전자정부 연계 API 이용지원을 위한 기능
  2. ② 1항에 따른 서비스 제공범위에 변동이 있는 경우에는 해당 내용을 포털에 사전 게시하여야 한다.

제5조(사용자 등록 및 개인정보 제공 동의)

  1. ① 각 기관 및 기업은 다음 각 호에 해당하는 담당자를 정하여 포털에 회원가입을 신청하여야 한다.
    1. 1. 총괄관리자 : 포털 운영에 관한 정책을 총괄하는 관리자로서 총괄기관(행정안전부)의 주무부서 담당자
    2. 2. 총괄운영자 : 포털 운영을 위탁받은 운영기관 담당자
    3. 3. 정보보유기관 담당자 : 전자정부 연계 API에 행정정보를 제공하는 원천기관 담당자
    4. 4. 이용기관 담당자 : 민간 SaaS를 이용하는 행정기관, 공공기관 및 교육기관 담당자
    5. 5. 기업 담당자 : 전자정부 연계 API를 이용하는 민간 SaaS 기업 담당자
  2. ② 민간 SaaS 기업은 기업 담당자 중 1인을 대표관리자로 지정하여, 전자정부 연계 API 포털에 등록된 담당자를 현행화하여야 한다. 특히 운영기관의 통지를 수신할 수 있도록 전자우편주소 및 휴대전화번호 등 포털에 등록된 업무담당자의 연락처 정보를 항상 최신으로 유지하여야 한다.
  3. ③ 사용자는 회원가입 시 성명, 소속기관(기업) 및 부서, 직급 및 본인인증이 필요한 기관(기업) 소속 이메일, 유선 전화번호, 개인 휴대번호 등 서비스 제공과 유사시 대응에 필요한 개인정보를 총괄기관(행정안전부장관) 및 운영기관이 포털을 통해 수집하여 처리하고 보유할 수 있음을 동의하여야 한다.
  4. ④ 제3항에 따라 수집된 담당자 정보는 사용자 변경, 담당자 교체, 연락처 변경 등으로 갱신되는 경우에도, 제3항에서 정한 목적(포털 서비스 제공, 이용기관·정보 보유기관·민간 SaaS 기업·운영기관 간의 연락 및 장애·사고 대응)을 위하여 동일한 범위 내에서 계속하여 이용·보유될 수 있다.

제6조(전자정부 연계 API 이용신청)

  1. ① 민간 SaaS 기업은 전자정부 연계 API를 이용하기 위해서는 민간 SaaS 기업의 기본정보와 이용방식 등을 포털 이용신청 절차에 따라 기재하여 운영기관에 제출하여야 한다.
  2. ② 제1항에 따른 이용신청을 받은 운영기관은 신청받은 내용을 확인 후 승인하고 제출받은 정보를 포털에 등록하여야 한다.
  3. ③ 제2항에 따른 등록 후 기업 담당자는 최초로 전자정부 연계 API에 접속하여 연락처와 전자우편주소 등 추가정보를 입력하고 본인인증을 하여야 한다.
  4. ④ 전자정부 연계 API 이용신청 절차에 관한 상세 내용은 포털에 게시된 이용안내 매뉴얼을 참조한다.

제7조(Mock API Test KEY와 운영 KEY 발급)

  1. ① 민간 SaaS 기업은 전자정부 연계 API 이용신청이 승인된 이후 포털에 민간 SaaS 기업 서버 정보 및 IP 정보 등을 입력하고 Mock API Test KEY 발급을 신청한다.
  2. ② 민간 SaaS 기업은 발급받은 Mock API Test KEY를 사용하여 전자정부 연계 API 민간 SaaS 기업 인증절차를 수행한 후 운영 KEY을 발급받을 수 있다.
  3. ③ 아래 각 호의 경우에는 기존에 발급받은 운영 KEY를 폐기하고 재발급받은 신규 운영 KEY를 사용하여야 한다.
    1. 1. 사업자번호가 변경된 경우
    2. 2. 외부 노출이나 오남용 등의 이유로 기존에 발급받은 운영 KEY를 폐기해야 하는 경우

제8조(전자정부 연계 API 이용시간 및 운영원칙)

  1. ① 운영기관은 연중무휴, 1일 24시간(00:00-24:00)으로 전자정부 연계 API를 운영하는 것을 원칙으로 한다. 다만, 다음 각 호의 경우 전자정부 연계 API의 전부 혹은 일부에 대한 운영이 중단 또는 제한될 수 있으며 그 사유가 해소되는 즉시 운영을 재개하여야 한다.
    1. 1. 전자정부 연계 API 시스템의 정기점검 또는 증설 및 교체, 새로운 API 기능 적용 등을 위하여 필요한 경우
    2. 2. 국가비상사태, 정전 및 그에 준하는 상황이 발생한 경우
    3. 3. API 서비스 이용의 폭주 등으로 시스템에 장애가 발생한 경우
    4. 4. 그밖에 전자정부 연계 API 운영 업무상 또는 기술상 사유로 전자정부 연계 API를 운영하기 어려운 상황이 발생한 경우
  2. ② 운영기관은 제1항 제1호에 따라 서비스 제공을 중단하거나 이용을 제한하는 경우에는 중단 및 제한 1주일 전, 제1항 제2호 내지 제4호에 따라 서비스 제공을 중단하거나 이용을 제한하는 경우에는 그 즉시 민간 SaaS 기업에 대해 중단 혹은 제한의 사유 및 기간 등에 관한 내용을 통지하여야 한다.

제9조(전자정부 연계 API 이용제한)

  1. ① 운영기관은 민간 SaaS 기업이 제13조의 의무를 위반한 경우 전자정부 연계 API 제공을 중단 또는 제한 할 수 있으며, 이 경우 해당 민간 SaaS 기업에 그 제한에 관한 내용을 제한 시작 30일 전에 통지하여야 한다.
  2. ② 운영기관은 제5조제1항에 따른 등록 후 6개월 이상 이용기관에 서비스 제공을 지원한 기록이 없는 민간 SaaS 기업에 대하여 전자정부 연계 API와의 연계를 제한할 수 있다.
  3. ③ 민간 SaaS 기업은 제1항과 제2항에 따른 제한 조치에 대한 이의신청을 할 수 있으며, 운영기관이 이의신청이 정당하다고 인정하는 경우 민간 SaaS 기업은 전자정부 연계 API 이용을 재개할 수 있다.

제10조(전자정부 연계 API 이용해지 및 중지)

  1. ① 민간 SaaS 기업은 서비스 기간의 종료 또는 제13조에 따른 민간 SaaS 기업의 의무 위반 등 사정이 없는 이상 전자정부 연계 API에서 제공하는 API를 이용하여 전자정부 연계 API의 기능을 계속 이용할 수 있다.
  2. ② 이 약관에서 정하지 아니한 사항은 「클라우드컴퓨팅 발전 및 이용자 보호에 관한 법률」 과 동법 시행령 및 시행규칙, 「행정기관 및 공공기관의 클라우드컴퓨팅서비스 이용 기준 및 안전성 확보 등에 관한 고시」 등 관계 법령을 준용한다.
  3. ③ 그밖에 이 약관 및 관계 법령에서 정하지 아니한 사항은 행정안전부장관과 민간 SaaS 기업의 장이 협의하여 정한 바에 따른다.
  4. ④ 휴면계정으로 전환된 회원이 전환일로부터 3년 이내에 포털에 로그인하거나, 보안성 검토 부적격 사유를 보완하여 API 재이용 요건을 충족한 경우에는 정상회원으로 복구한다. 다만, 휴면계정 전환 후 3년간 로그인 또는 보완이 이루어지지 않은 경우에는 회원탈퇴 처리한다.

제11조(서비스 장애 및 일시중지)

  1. ① 정보보유기관과 민간 SaaS 기업은 API 연계 또는 서비스의 장애, 지연 현상이 발생한 경우 즉시 운영기관에 알려야 한다.
  2. ② 정보보유기관과 민간 SaaS 기업은 시스템의 정기점검 또는 증설 및 교체, API 변경 적용 등으로 제공 업무의 전부 또는 일부를 일시적으로 중지하고자 하는 경우 최소 1주일 전에 운영기관 및 이용기관에 알리고, 기업의 웹 또는 앱 등을 통하여 공지하여야 한다.

제12조(정보의 반환 및 파기 등)

  1. ① 민간 SaaS 기업은 이용기관과의 계약의 해제, 해지 또는 서비스 기간 만료 등으로 서비스가 종료되는 경우에는 민간 SaaS 기업의 정보자원에서 관리되는 정보를 다음 각 호 중 어느 하나의 방법으로 조치해야 한다.
    1. 1. 이용기관에 처리 데이터를 반환
    2. 2. 이용기관이 반환을 받지 않거나 반환이 불가능한 경우에는 복구가 불가능한 방법으로 지체 없이 파기
    3. 3. 이용기관이 지정한 다른 서비스 제공자에게 이관하고, 민간 SaaS 기업이 보유한 저장정보를 복구가 불가능한 방법으로 지체 없이 파기
  2. ② 이용기관이 제1항에 따라 처리정보를 반환받는 경우에는(이용기관의 장을 대신 하여 다른 서비스 제공자가 저장정보를 이관받는 경우 포함) 다음 각 호의 방법에 따른다.
    1. 1. 온라인으로 반환받는 경우, 저장정보를 암호화하고 전용선, 가상사설망 등 안전한 경로를 이용하여 전송
    2. 2. 오프라인으로 반환받는 경우, 저장정보를 암호화하여 저장매체에 저장하고 해당 저장매체가 탈취되지 않도록 보호하여 이송
    3. 3. 제1호 또는 제2호에 따라 반환받는 저장정보는 반환 즉시 활용이 가능한 형태로 전송 또는 이송
  3. ③ 정보의 반환 및 파기에 관해 동 약관에서 규정하지 아니한 사항은 ｢행정기관 및 공공기관의 클라우드컴퓨팅서비스 이용 기준 및 안전성 확보｣ 등에 관한 고시에 따른다.

제3장 운영기관 및 민간 SaaS 기업의 의무 등

제13조(운영기관의 의무)

  1. ① 운영기관은 전자정부 연계 API를 통한 API 서비스를 지속적이고 안정적으로 제공하기 위하여 노력하여야 한다.
  2. ➁ 운영기관은 API 서비스 관련 불만 사항이 접수되는 경우 이를 신속하게 처리하여야 하며 신속한 처리가 곤란한 경우 그 사유와 처리 일정을 포털을 통해 공지하여야 한다.
  3. ③ 운영기관은 전자정부 연계 API와 민간 SaaS 기업의 API 연계 서버가 정상 상태인지 모니터링하고 관련 정보를 제공하여야 하며, 정상 상태가 아닌 경우 신속하게 복구하여야 한다.
  4. ④ 운영기관은 정보보유기관으로부터 서비스 중단 사유 발생, 서비스 제공 시 공지한 제반 사항과 API 서비스 이용에 관한 변경사항 등을 통지 또는 통보받는 경우, 이를 민간 SaaS 기업에 신속하게 통보해야 한다.

제14조(민간 SaaS 기업의 의무)

  1. ① 민간 SaaS 기업은 전자정부 연계 API를 이용함에 있어서 이 약관에서 규정하는 사항과 관계법령을 준수하여야 한다.
  2. ② 민간 SaaS 기업은 전자정부 연계 API를 이용기관에게 제공함에 있어서 다음과 같은 의무를 부담한다.
    1. 1. API 서비스 업무담당자를 지정하고 이를 운영기관에게 통지한다.
    2. 2. 발급받은 API Key를 선량한 관리자의 주의의무로 관리하고 타인에게 제공, 양도하거나 공개, 공유하여서는 아니된다.
    3. 3. 발급받은 API Key가 타인에 의하여 도용당하지 않도록 관리하여야 한다.
    4. 4. 타인이 발급받은 API Key를 이용하여 전자정부 연계 API에 접근해서는 안된다.
    5. 5. 정보주체의 동의나 정보보유기관과 체결한 별도의 협약이 없는 이상 API 서비스 제공과 관련하여 취득한 정보의 임의 저장, 제3자 제공 및 재가공 하여서는 아니된다.
    6. 6. 운영기관 또는 정보보유기관이 요청하는 경우 API 서비스의 이용 통계 정보를 제공하여야 한다.

제15조(통지의 방법)

  1. ① 운영기관은 민간 SaaS 기업에 대한 통지가 필요한 경우 민간 SaaS 기업의 전자정부 연계 API 이용신청 시 등록된 업무담당자의 전자우편 혹은 휴대전화 및 기타 방법을 통해 통지할 수 있다.
  2. ② 불특정 다수 이용기관에 대한 통지의 경우 포털에 게시함으로써 개별 통지를 갈음할 수 있다.

제16조(개인정보 보호 등)

  1. ① 운영기관이나 민간 SaaS 기업이 전자정부 연계 API의 이용과 관련하여 이용기관의 개인정보를 처리하는 경우 「개인정보 보호법」 , 「정보통신망 이용촉진 및 정보보호 등에 관한 법률」 등 관련 법령에 따라 처리하여야 한다.
  2. ② 민간 SaaS 기업은 전자정부 연계 API 이용신청 시 운영기관에 제출한 기본정보 등 민간 SaaS 기업에 관한 정보는 민간 SaaS 기업이 API 서비스 제공 지원을 중단하거나 해지할 때까지 전자정부 연계 API에 보관하여야 한다.

제4장 사고 및 장애 대응

제17조(사고 및 장애에 대한 대응계획 수립 등)

  1. ① 운영기관은 다음 각 호의 어느 하나에 해당하는 경우 포털 서비스의 전부 또는 일부를 일시적으로 또는 영구적으로 중단할 수 있다.
    1. 1. 국가비상사태, 정전, 설비 장애, 기타 불가항력적인 사유로 포털 서비스의 원활한 제공이 어려운 경우
    2. 2. 관련 법령에 근거하여 업무절차 또는 시스템의 개선이 필요한 경우
    3. 3. 그 밖에 포털 서비스의 계속적인 제공이 현저히 곤란하다고 판단되는 경우
  2. ② 운영기관이 포털 서비스를 영구적으로 중단하고자 하는 경우에는 중단 예정일 3개월 전까지 또는 중단 사유가 발생한 사실을 인지한 때에는 지체 없이 회원에게 이를 공지하여야 한다.
  3. ③ 민간 SaaS 기업은 다음 각 호의 사고에 대비한 대응계획 및 처리방안을 수립하여야 한다.
    1. 1. 전자정부 연계 API 또는 포털과 연계된 전산기기 또는 통신회선 등의 장애 또는 가동중단으로 인하여 이용기관에 대한 API 서비스가 중단되는 경우
    2. 2. API 서비스 제공과 관련된 전산보안시스템이 작동불능의 경우
    3. 3. API 서비스 제공과 연관된 정보가 누출되거나 자체 시스템에 대한 해킹이 발생한 경우
    4. 4. 그 밖의 사유로 API 서비스 제공 업무에 장애가 발생한 경우
  4. ④ 민간 SaaS 기업은 정상적인 API 서비스 제공에 문제가 발생한 경우 그에 대한 내용을 즉시 운영기관에게 통보하여야 한다.
  5. ⑤ 정보보유기관은 전자정부 연계 API 또는 자체 정보시스템의 장애, 보안사고 등으로 인하여 연계 API 제공에 중대한 지장이 발생하거나 발생이 예상되는 경우, 지체 없이 그 사실 및 조치계획을 운영기관에 통보하여야 하며, 이용자 피해 최소화를 위해 필요한 조치를 즉시 취하여야 한다.
  6. ⑥ 정보보유기관은 제5항에 따른 장애·사고의 재발 방지 및 영향 최소화를 위하여 관련 로그, 점검결과, 재발 방지 대책 등을 운영기관에 지체 없이 제출하여야 하며, 운영기관이 민간 SaaS 기업 및 이용기관에 안내할 수 있도록 사고 원인 규명에 적극 협력하여야 한다.

제5장 저작권 등

제18조(저작권의 귀속 등)

민간 SaaS 기업에게 제공하는 각 정보별 API 등에 대한 저작권은 행정안전부 또는 정보보유기관에 있다.

제19조(책임의 제한)

  1. ① 행정안전부장관은 전자정부 연계 API 이용에 관하여 본 약관, 제반 규정, 이용 안내 및 관계 법령을 준수하지 않은 이용으로 인한 결과에 대하여 책임을 지지 않는다.
  2. ② 행정안전부장관은 제8조제2항에 따른 전자정부 연계 API 운영 중단이나 제한으로 인하여 민간 SaaS 기업이 API 서비스를 이용하지 못하게 되거나 메시지나 통신 데이터가 손실되거나 전송되지 못하여 이용기관이나 민간 SaaS 기업에 손해가 발생한 경우라도 행정안전부의 고의나 과실이 없는 한 그에 대한 책임을 부담하지 아니한다.
  3. ③ 행정안전부장관은 API 서비스 제공과 관련하여 정보보유기관 또는 운영기관의 책임없는 사유로 민간 SaaS 기업과 이용기관 사이에 발생한 분쟁에 대하여 개입할 의무가 없으며 이로 인한 손해를 배상할 책임이 없다.
  4. ④ 한국지능정보사회진흥원(NIA) 및 운영기관은 제17조에 따라 서비스가 중지되는 사유와 관련하여 회원 또는 제3자에게 손해가 발생하더라도 책임을 지지 않는다. 다만 해당 손해가 한국지능정보사회진흥원(NIA) 및 운영기관의 고의로 행한 범죄 행위를 제외하고 이에 대하여 책임을 부담하지 않는다.

제20조(약관 외 준칙)

  1. ① 행정안전부장관은 필요한 경우 정보보유기관 및 민간 SaaS 기업과 협의하여 특정 API 서비스에 관하여 적용될 사항을 정한 별도의 전자정부 연계 API 이용정책을 운영할 수 있다.
  2. ② 이 약관에서 정하지 아니한 사항은 행정안전부장관과 민간 SaaS 기업의 장이 협의하여 정한 바에 따른다.

제21조(준거법 및 재판관할)

  1. ① 전자정부 연계 API의 이용에 관하여 발생한 분쟁에 관한 소송은 대한민국 법령을 준거법으로 한다.
  2. ② 전자정부 연계 API의 이용에 관하여 발생한 분쟁에 관한 소송은 대한민국 서울중앙지방법원을 전속관할로 한다.
